# 🧪 Book API — QA Portfolio

Projeto de API REST desenvolvido com foco em Qualidade de Software (QA), testes e validação de regras de negócio.

## 📌 Sobre o Projeto

Esta aplicação simula um sistema de gerenciamento de livros e avaliações, permitindo validar o comportamento da API em diferentes cenários.

O projeto foi estruturado para demonstrar, na prática, habilidades em testes de software, organização de cenários e análise de erros.

## 🎯 Objetivo

- Aplicar conceitos de QA em um projeto real
- Validar regras de negócio da API
- Criar e documentar casos de teste
- Simular cenários positivos e negativos

## 🧪 Estratégia de Testes

Os testes foram organizados para cobrir diferentes tipos de cenários:

- ✔ Testes funcionais (fluxos principais)
- ✔ Testes negativos (validação de erros)
- ✔ Testes de API (status e comportamento)
- ✔ Validação de regras de negócio

## 📁 Estrutura de Testes

test-cases/
├── functional/
├── negative/
├── api/

## 🧠 Abordagem

- Testes baseados no comportamento esperado da API  
- Validação de status codes (200, 201, 400, 401)  
- Verificação de entradas inválidas  
- Organização clara e reutilizável  

## 📊 Exemplos de Cenários

- Criar livro com dados válidos  
- Criar livro sem título  
- Login com dados inválidos
